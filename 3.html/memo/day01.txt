인터넷
	넷.네트워크
	전 세계에 걸쳐 원거리 접속이나 파일 전송, 전자 메일 등
	데이터 통신 서비스를 받을 수 있는 컴퓨터 네트워크 시스템

클라이언트와 서버
	클라이언트(정보를 제공받는 사용자 또는 컴퓨터)
		요청 ↓	↑ 응답
	서버(정보를 제공하는 사용자 또는 컴퓨터)

프로토콜(Protocol)
	넷 상에서 컴퓨터끼리 소통할 때 사용해야 하는 규칙들을 프로토콜이라 한다. 
	통신하기 위한 필요한 규약

IP(Internet Protocol)
	네트워크 상에서 다른 컴퓨터와 구별될 수 있도록 고유번호(주소)를 부여하는 프로토콜 000~255 * 4 (IPv4) 숫자 4개를 이용해서 번호를 부여했음

HTTP(HyperText Transfer Protocol)
	클라이언트와 서버간의 웹 페이지 등의 자원을 통신하는 규약

FTP(File Transfer Protocol)
	클라이언트와 서버간의 파일 전송을 위한 규약

HyperText
	한 문서에서 다른 문서로 접근하는 개념
	문서에 주소(URL)를 붙이고 그 주소를 통해 이동, 요청 하는 것을 HyperText라 부른다. 

웹 브라우저
	사용자의 요청에 맞는 주소를 찾아가서 인터넷의 컨텐츠(문서와 그림, 멀티미니어 파일 등)를 검색 및 열람 후 
	사용자에게 보여주기 위한 응용 프로그램의 총칭
	즉, 인터넷을 보여주기 위해 브라우저를 통함.

도메인(Domain)
	IP 주소는 기억하고 이해하기 힘들기 때문에 이를 위해 
	이름을 부여할 수 있도록 하는 서비스(DNS, Domain Name Servie)를 통해 받은 이름

HTML(HyperText Markup Language)
	텍스트로 페이지의 구조와 데이터들을 표기하는 언어
	HTML 문서는 확장자가 .html로 문서를 작성할 수 있는 프로그램이라면 어떠한 것이든지 작성이 가능하다.(메모장도 가능)
	대소문자를 구별하지 않는다.
	띄어쓰기, 줄바꿈을 구별하지 않는다.
	태그 형태로 꺽쇠괄호(<명령어>)를 사용하여 표현한다.
	
HTML의 요소(태그)
	①<p>③웹과의 첫 만남②</p>
	
	① 여는 태크(Opening Tag)	: 요소의 이름(p)이 열고 닫는 꺽쇠 괄호 안에 있음
	② 닫는 태그(Closing Tag)	: 여는 태그와 똑같이 생겼지만, 요소의 이름 앞에 슬래시(/)가 있음
	③ 내용(Content)		: 요소의 내용이며, 단순한 텍스트 혹은 다른 태그
