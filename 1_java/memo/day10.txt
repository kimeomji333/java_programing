HashSet
	Hash는 해시값, Set은 집합이라는 뜻이다.
	해시값을 이용한 집합
	집합은 중복되는 요소(원소, 값)를 포함할 수 없다.
	저장된 요소들에는 인덱스가 없기 때문에 저장 순서를 따로 관리하지 않는다.
	-> 값의 포함 여부, 값의 유무 검사 목적이 있다. 
	Set은 검색의 목적이 있기 때문에 순서 정보를 관리할 필요가 없다. 
	
	-순서가 없음? 순서 정보를 관리할 필요 없다.
	-중복되는 원소를 담을 수 없다.

HashMap
	해시값을 이용해서 지도처럼 저장을 함
	Map 구조는 key(키)와 value(값)가 한 쌍으로 저장되는 형태이다.
	검색에 용이하다.
	key는 중복이 불가능하며, value는 가능하다. 
	key는 Set타입에 저장되고, value는 Collection 타입에 저장된다.
	키 <-> 해시 테이블 <-> 값

	key	value
Entry	fly	날다 - 이러한 한 쌍을 Entry라고 한다.
Entry	run	뛰다

	.keySet()	:key들만 모여있는 Set을 리턴
	.values()	:value들만 모여있는 Collection을 리턴
	.entrySet()	:Entry들이 모여있는 Set을 리턴

컬렉션 이용시 주의할 점
	컬렉션에서는 요소들을 비교할 때 그 요소 타입의 equals() 혹은 hashCode()로 비교하기 때문에
	우리가 만든 클래스에서 그것들을 재정의하지 않는다면 Object 클래스에 있는 주소값 비교를 하기 때문에 같은 요소를 찾을 수 없게 된다. 
	따라서 비교하고자 하는 요소들을 올바르게 비교하도록 두 메소드를 "재정의"해서 컬렉션을 사용해야 한다. 

향상된 for문(빠른 for문, forEach문)
	반복횟수가 특정된...
	컬렉션 안의 요소들의 개수가 정해져 있지 않아, 단순하게 모든 값들을 순회할 때 사용하는 것이 좋다.
	순서가 중요한 경우에는(몇 번째에 어떤 값이 있는지 알아야 할 필요가 있는 경우) 기존의 for문을 이용하는 것이 좋다.

	for(자료형 변수명 : 반복자){
		변수가 안에 있는 요소들을 하나씩 꺼내오며 반복
	}

API(Application Programming Interface)
	어플리케이션 프로그래밍에 필요한 것들을 통틀어서 모아놈 ex)JRE
