통신: 서로 연결되어 대화를 나눈다.

http 통신
	클라이언트(사용자)와 서버(백엔드 API)가 요청과 응답이라는 것을 통해 하는 통신
	요청 없이는 응답이 이루어지지 않음

소켓 통신
	http 하면서 특수한 상황에서 불편한 상황(ex) 채팅)에 소켓을 열어놓고, 요청이 없더라도 통신을 받을 수 있음
	클라이언트(사용자)와 서버(백엔드 API)가 특정한 통로(port)를 통해 하는 통신
	실시간으로 이루어지는 양방향 통신에 해당함
	ex) 실시간 채팅, 스트리밍 등이 소켓 통신을 사용하는 대표적인 기술  
	
	1. 클라이언트가 서버의 주소(ip, address..)로 특정 port 번호를 사용해서 연결을 요청 - (악수/handshake)
	2. 요청을 받은 서버는 연결을 허용하며 클라이언트의 소켓과 연결
	3. 실시간으로 통신 가능한 네트워크 통로가 열렸으니 언제든 통신 가능
		서로간의 실시간 통신이 가능하도록 계속해서 요청을 듣고(listen) 있는 상태
		통신의 특성상 서로 대규모의 데이터는 전송하지 않고 작은 데이터 위주로 통신
		양쪽에서 약속해놓은 형태의 데이터(규격화된 문자열(JSON))를 서로 보내고, 그 데이터를 해석 후 사용
