프로그램 
	잘 짜여진 틀

Thread(쓰레드)
	프로세스의 특정한 수행 경로
	프로그램의 처리 경로(흐름)
	
멀티 쓰레드
	여러개의 작업을 동시에 처리하는 것처럼 보이지만 사실은 매우 짧은 단위로 분할해서 차례대로 처리한다.
	여러개의 처리 경로를 가질 수 있다.
	한 개의 프로세스를 여러개로 나누어 동시 작업이 가능해진다.
	웹 서버 프로그램(JSP)이 대표적인 멀티 스래드 응용 프로그램이다.

장점
	효율성 증가
	처리량 증가
	처리비용 감소
단점
	복잡하고 설계가 어려움
	자원 공유 문제
	교착상태(deadlock)
	* 교착상태: 멀티 쓰레드 중 쓰레드 간에 대기 상태가 종료되지 않아서 무한정 대기만 하는 비정상적인 상태
	대비 방법 -> 제어문을 통해 교착상태를 판단하여 다시 깨워준다.

동기화(synchronized)
	<- 하나의 공유자원을 여러 쓰레드에서 동시에 접근하게 되면 때때로 우리가 예상치 못한 결과가 나타나게 되는게 이러한 문제를 해결하기 위해 사용됨
	하나의 쓰레드가 작업 중일 때 다른 쓰레드가 동시에 같은 필드(자원)를 사용하지 못하도록 막는 것

쓰레드 생성
	1. Thread 클래스 상속
	2. Runnable 인터페이스 상속 후 Thread 객체에 넘겨주기


파일 입출력
	파일: 일정 영역의 공간을 확보해서 1010 이라는 데이터를 쓰게 되면 파일이게 됨. (데이터가 없이 공간만 확보되어도 파일)
	파일(데이터의 집합체) 에서 데이터를 가지고 온다. 
	
	디렉토리 서비스
	외부 파일		->	자바 프로그램
	(데이터)			10101010...라고 읽어 옴

		input/output 스트림(데이터를 읽고 쓰고 하는 기본적 클래스)


	Writer(쓰기, 입력)
		FileWriter > BufferdWriter > 파일에 쓰기

		FileWriter: 경로에 있는 파일을 자바 시스템으로 가져오기(쓰기 위해)
		*파일이 없는 경우에는 그 이름대로 파일을 생성해준다. 
		*경로가 이상한 경우(폴더가 잘못된 폴더인 경우)에는 폴더를 생성해 주지는 않고, 예외를 발생시킨다.

		new FileWriter("파일경로")		: 덮어쓰기 모드
		new FileWriter("파일경로", true)	: 이어쓰기 모드

		BufferdWriter: 버퍼를 이용해서 파일 작성
		bw.write("문자열")
		bw.close()			: 실제 파일에 적용

	Reader(읽기, 출력)
		FileReader > BufferdReader > 파일에서 읽기

		FileReader: 경로에 있는 파일을 자바 시스템으로 가져오기(읽기 위해)
		*파일이 존재하지 않으면 예외 발생(FNFE, File Not Found Exception)

		BufferedReader: 버퍼를 이용해서 파일 읽기
		br.readLine()			: 파일에 있는 내용을 한 줄 이동하며 읽어오기

	파일 복사, 다운로드를 예시로 들어보자.

	- 파일은 데이터에 해당. 실존
	- 폴더는 데이터가 아니기에 실존하지 않음. 구획 같은 느낌

공부해야 할 키워드
- 스트림: (간단하게 데이터의 흐름) 자바에서는 파일이나 콘솔의 입출력을 직접 다루지 않고, 스트림이라는 흐름을 통해 다룬다.
- 패킷: 네트워크를 통해 전송되는 형식화된 데이터 덩어리(데이터의 기본 전송 단위, 데이터가 한 묶음씩 이동하는 기준)
- 버퍼: 데이터를 한 곳에서 다른 한 곳으로 전송하는 동안 일시적으로 그 데이터를 보관하는 메모리의 영역
