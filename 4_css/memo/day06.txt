미디어 쿼리
	width, height, color,... 속성과 같은 미디어 관련 표현식을 통해
	스타일이 적용되는 범위를 조절하는 기법
	반응형 개발
	컨텐츠를 별도로 변경하지 않아도 웹 페이지에 접속하고 있는 기기에 따라
	알맞은 형태로 스타일을 조정

	장점 : UI 고급화
	단점 : 다운로드 시간이 걸림, 구현이 어렵다.

	기본 스타일링 코드
	<style>
		선택자{
			속셩명 : 속성값;
		}
	</style>

	미디어 쿼리
	<style>
		@media 매체 유형 and 조건식 .. {
			선택자{
				속성명 : 속성값;
			}
		}
	</style>

	매체 유형
		all	: 모든 매체
		screen	: 컴퓨터, 테블릿, 스마트폰, ..
		print	: 프린터 기기
		speech	: 스크린 리더기
		(하지만 대부분 all 아니면 screen 을 사용한다)

벤더 프리픽스(vendor prefix)
	주요 웹 브라우저 공급자가 새로운 실험적인 기능을 제공할 때
	이전 버전의 웹 브라우저에 그 사실을 알리기 위해 사용하는 접두사를 의미
	아직 css 권고 안에 포함되지 않은 기능이나, 포함되어 있지만
	아직 완벽하게 제정된 상태가 아닌 기능을 사용할 때 앞에 붙여준다.

	벤더 프리픽스를 사용하는 방법은 해당 속성이 적용되지 않았을 경우
	표현해야 하는 코드를 가장 먼저 작성하고, 표준 문법 코드는 가장 마지막에 작성해야 한다.
	그 둘 사이에 벤더 프리픽스를 이용한 코드를 작성해준다.

	브라우저가 인식할 수 있도록 -webkit, -moz, -o, .. 등등 을 (브라우저 판매사를) 앞에 적어준다.
	벤더 프리픽스를 적어준 background를 먼저 적고 나중에 표준코드를 적어준다.

transition
	요소에 추가할 css 스타일의 전환효과를 설정
	추가될 전환효과나 지속될 시간도 설정

	transition:property timing-function duration delay;

	property	: 요소에 추가할 전환효과 중 속성을 작성
	timing-function	: 전환효과의 변화되는 속도를 설정
			linear - 일정한 속도
			ease - 전환효과의 속도가 달라지도록 설정
			(ease-in, ease-out, ease-in-out)
	duration	: 전환효과의 시간 설정
	delay		: 전환효과의 대기시간 설정, duration + delay = 전체 시간


CSS 애니메이션
	요소의 현재 스타일을 다른 스타일로 변화시키며 애니메이션 효과를 낼 수 있다.
	
	@keyframes 애니메이션명{
		0% 혹은 from {
			처음에 적용될 스타일
		}
		
		정수% {
			중간지점 마다 적용할 스타일
		}

		100% 혹은 to {
			마지막에 적용할 스타일
		}
	}
