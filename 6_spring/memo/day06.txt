# Model
   redirect 방식은 Model을 들고 가지 못한다.
   주소를 알려주고 재요청하는 것이기 때문이다.
   
   그렇다고 redirect 방식에서 데이터를 못 들고 가는 것이 아니다.
   아에 다른 곳(세션 등)에 데이터를 넣고, 다시 가져오는 방식을 사용한다.

# 서버사이드렌더링, 클라이언트 사이드 렌더링의 차이
   클라이언트사이드렌더링 - CSR
      3번 게시글 주세요. -> 일단 페이지 이동 -> 서버와 통신
       -> POST로 넘기고, POST 받아서 AJax(비동기)로 데이터를 화면에 입력.
       
   그래서 서버를 2개 돌려야 한다.
   브라우저 ~> 프론트~> 서버
   
   무한스크롤 - Ajax로도 해볼 수 있다. 새로운 데이터를 비동기로 가져오기.
